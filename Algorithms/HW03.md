## Problem 2
### (a)
### Basic Idea
Use **low** and **high** pointers to track indices in the array.  
Compare `array[(low + high) // 2]` with `(low + high) // 2 + 1`:  
- If equal, set `low = (low + high) // 2 + 1`.  
- Otherwise, set `high = (low + high) // 2 - 1`.  

Continue iterating until `low < high`.  
The missing element will be **low + 1**.  
The runtime is $O(\log n)$.

### Pseudocode
```python
findMissing(array):
    low = 0
    high = array.len - 1
    while high >= low:
        if array[(low + high) // 2] == (low + high) // 2 + 1:
            low = (low + high) // 2 + 1
        else:
            high = (low + high) // 2 - 1
    return low + 1
```

## Problem 3

### Basic Idea

Finding the median of $m+n$ elements is equivalent to finding the  
$k$-th smallest element, where

- $k = \frac{m+n}{2}$.

Now compare the $\lfloor k/2 \rfloor$-th element in both arrays $A$ and $B$:

- If $A[\lfloor k/2 \rfloor] > B[\lfloor k/2 \rfloor]$,  
  then the elements $B[0], B[1], \dots, B[\lfloor k/2 \rfloor - 1]$  
  cannot be the $k$-th smallest element.  
  → Discard these elements, update $B = B[\lfloor k/2 \rfloor : \text{end}]$,  
  and recursively search for the $(k - \lfloor k/2 \rfloor)$-th smallest element.

- If $A[\lfloor k/2 \rfloor] \le B[\lfloor k/2 \rfloor]$,  
  then the elements $A[0], A[1], \dots, A[\lfloor k/2 \rfloor - 1]$  
  cannot be the $k$-th smallest element.  
  → Discard these elements, update $A = A[\lfloor k/2 \rfloor : \text{end}]$,  
  and recursively search for the $(k - \lfloor k/2 \rfloor)$-th smallest element.

- If the number of elements in one array is **less than $\lfloor k/2 \rfloor$**,  
  discard all elements from that array and reduce $k$ accordingly,  
  since all those elements are guaranteed to be smaller than the target.


#### Pseudocode

```python
# first k is [(m + n) / 2] + 1
GetKth(A, B, k):
    half_k = floor(k / 2)
    if A.len == 0: return B[k - 1]
    if B.len == 0: return A[k - 1]
    if k == 1: return min(A[0], B[0])
    i = min(A.len, half_k)
    j = min(B.len, half_k)
    if A[i - 1] < B[j - 1]:
        A = A[i to end]
        k -= i
    else:
        B = B[j to end]
        k -= j
    return GetKth(A, B, k)
```

## Problem 4

### (a)

#### Basic Idea
For a given player, query every other player exactly once. Count how many times the player is labeled **citizen** and how many times **werewolf**.  
- If `#werewolf > #citizen`, classify the player as a **werewolf**.  
- Otherwise, classify the player as a **citizen**.

This runs in time Θ(n).

#### Proof
- **Case 1: The player is a werewolf.**  
  Among the remaining players, citizens outnumber werewolves. Since citizens always tell the truth, each citizen will label the player a werewolf. Therefore, the number of **werewolf** votes strictly exceeds the number of **citizen** votes.

- **Case 2: The player is a citizen.**  
  - If citizens are the strict majority, they will correctly label the player a citizen, so **citizen** votes exceed **werewolf** votes.  
  - If citizens and werewolves are tied in number, citizens still label the player a citizen, so **citizen** votes are at least as many as **werewolf** votes.

Hence, the classification is correct in all cases.

### (b)
#### Basic Idea
Split the n players into two groups of size n/2. In at least one group, citizens still outnumber werewolves (the invariant). Recursively apply the procedure to each group to obtain two **citizen candidates**. Finally, verify both candidates using the method from (a) and return the one that passes.  
This algorithm runs in time O(n log n).
#### Pseudocode
```python
isCitizen(player) => bool # method in (a)
getCitizen(players):
    if players.len == 1:
        return players[0]
    else:
        a = getCitizen(player[from head to n/2])
        b = getCitizen(player[from n/2 + 1 to end])
        if isCitizen(a): return a
        else: return b
```
#### Proof
- **Base Case (n = 1).** The single player is correctly identified by the verifier in (a).  
- **Inductive Hypothesis.** Assume the procedure is correct for all sizes up to k.  
- **Inductive Step (n = k + 1).**  
  When splitting into two groups of ⌈(k + 1)/2⌉ and ⌊(k + 1)/2⌋, at least one group preserves the invariant that citizens ≥ werewolves, so it is solvable by the recursive call (by the hypothesis). Each recursive call yields a candidate, and verifying both by (a) ensures that any werewolf candidate is rejected while any true citizen candidate is accepted. Therefore the returned player is a citizen.  
  Thus, by induction, the algorithm is correct for all n.

**Induction Hypothesis.** Assume `getCitizen` is correct for all $n \le k$.

**Induction Step ($n = k + 1$).**  
In any round, pairs can be of three types and singletons (when $n$ is odd) of two types:
- Pairs: $(W,W)$, $(W,C)$, $(C,C)$.
- Singletons: $W$ or $C$.

Update rule:
- $(W,W)$ and $(W,C)$ are **discarded**; neither contributes survivors.
- From each $(C,C)$ pair, **exactly one citizen** survives.
- If there is a singleton:
  - A singleton $C$ **survives** to the next round.
  - A singleton $W$ is **discarded** (it cannot be paired to survive under the rule).

Now distinguish parity:
- **Even $k+1$.** There are no singletons. Since each $(C,C)$ pair yields one citizen and all pairs containing a werewolf are discarded, the survivors consist of exactly one player per $(C,C)$ pair, hence **citizens $\ge$ werewolves** holds among survivors.
- **Odd $k+1$.** There is exactly one singleton.  
  - If the singleton is $C$, then survivors include all one-per-$(C,C)$ citizens **plus** this $C$; again, **citizens $\ge$ werewolves**.  
  - If the singleton is $W$, it is discarded, so survivors come only from $(C,C)$ pairs; thus **citizens $\ge$ werewolves** still holds.

Therefore, after each round the next instance is strictly smaller and preserves the invariant “citizens are at least as many as werewolves.” By the induction hypothesis, the recursive call correctly returns a citizen. Hence `getCitizen` is correct for $n=k+1$, and by induction for all $n$. The total running time is $O(n)$.

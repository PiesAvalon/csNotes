### Step 1: Proof that the Algorithm Halts
The outer loop is a for loop, which clearly terminates after a finite number of iterations. In the inner loop, `i` decreases by one in each iteration; thus, in the worst case, `i` will decrease to zero, which will terminate the loop.

### Step 2: Proof of Correctness

**Outer Loop Invariant:** At the end of each iteration, the elements between `A[1]` and `A[j]` are sorted.

**Proof of Outer Loop Invariant:**

- **Base Case (j = 2):** After the first iteration, the first two elements `A[1]` and `A[2]` are in order.
- **Inductive Hypothesis:** Assume that after an arbitrary iteration, the elements between `A[1]` and `A[k]` are sorted.
- **Inductive Step:** In the next iteration, `j = k + 1`. The inner loop starts with `key = A[k+1]` and `i = k`. Assume that at the end of each iteration of the inner loop, the elements between `A[i+1]` and `A[j]` are greater than `key` (this will be proved below as the inner loop invariant). When the inner loop halts, since it does not change the relative order of the previous `A[1]` to `A[j-1]` elements, the elements between `A[1]` and `A[i]` are less than `key`. At the end of the outer loop, the `key` element will be inserted in the correct location, and the elements between `A[1]` and `A[k+1]` will be sorted. Thus, the algorithm is correct.

**Proof of Inner Loop Invariant:**  
At the end of each iteration of the inner loop, the elements between `A[i+1]` and `A[j]` are greater than `key`.

- **Base Case:** In the first iteration where `i = j - 1`, if the loop halts here, the case is trivial. If not, this iteration ends with `i` decreased and `A[i+1]` and `A[j]` greater than `key`.
- **Inductive Hypothesis:** Assume that after an arbitrary iteration, the elements between `A[i+1]` and `A[j]` are greater than `key`.
- **Inductive Step:** In the next iteration, `A[i+1]` is shifted left to `A[i]`, and `i` is decreased by one. Thus, the elements between `A[i+1]` and `A[j]` remain greater than `key`.

Thus, the inner loop invariant is proved. Note that the inner loop does not modify the relative order of the sorted elements from previous iterations.

## Problem 6

A stack consists of two queues, both initialized as empty, and an integer counter set to 0.

- The queue containing elements is called the **main queue**; the empty one is the **helper queue**. If both are empty, either queue can be considered the main queue.
- **PUSH operation:**  
  - Increment the counter by one.
  - ENQUEUE the new element into the main queue.
- **POP operation:**  
  - DEQUEUE (counter - 1) elements from the main queue and ENQUEUE them into the helper queue in the same order.
  - Decrement the counter by one.
  - DEQUEUE and return the last element from the main queue.
  - Swap the roles of the main and helper queues.

**Running Time:**  
- `PUSH`: $\Theta(1)$  
- `POP`: $\Theta(n)$

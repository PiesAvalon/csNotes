## Problem 1

### (a)
Use a tree-shaped DP. Pick any node as the root to make the tree rooted.  
For each node maintain $m$ and $n$:

- $m[u]$ is the maximum path in $u$'s subtree that **starts at $u$**.  
  If $u$ is a leaf, $m[u] = 0$; otherwise
  $$
  m[u] = \max\bigl(0,\; w(u,v) + m[v]\bigr)
  $$
  over all children $v$ of $u$.

- $n[u]$ is the maximum path in $u$'s subtree that **passes through $u$** (may not start at $u$):
  $$
  n[u] = \max\Bigl(m[u],\; w(u,v) + m[v] + w(u,x) + m[x]\Bigr)
  $$
  for two distinct children $v, x$.

Each $m[u]$ and $n[u]$ can be computed in $O(1)$ time once child values are known.  
A DFS over all nodes is $O(n)$. Taking the maximum $n[u]$ adds another $O(n)$,  
so the total time complexity is $O(n)$.

### (b)
Use a similar tree DP. After rooting, for each node keep $root\_max$ and $total\_max$:

- $root\_max[u]$ is the best subtree containing $u$:
  $$
  root\_max[u] = \max\Bigl(0,\; u + \sum_{\substack{v \text{ child} \\ root\_max[v] > 0}} root\_max[v]\Bigr).
  $$

- $total\_max[u]$ is the best subtree entirely within $u$'s subtree:
  $$
  total\_max[u] = \max\Bigl(root\_max[u],\; \max_{v \text{ child}} total\_max[v]\Bigr).
  $$

The DFS runs in $O(n)$ time, and the final answer is $total\_max[root]$.

## Problem 2
For the string $T$, calculate the reverse string $R$.

Let $dp[i][j]$ be the maximum length of the same substring ending at $T[i]$ and $R[j]$.  
If $T[i] \ne R[j]$, then $dp[i][j] = 0$; otherwise:

$$
dp[i][j] = dp[i-1][j-1] + 1.
$$

To ensure the two occurrences in $T$ do not overlap, map index $j$ in $R$ back to $T$ by:

$$
bStart = n - j + 1.
$$

For $L = dp[i][j]$, the maximum valid (non-overlapping) length contributed by this pair is:

$$
best =
\begin{cases}
L, & i < bStart,\\[6pt]
\min\!\left(L,\; \left\lfloor \dfrac{i - bStart + 1}{2} \right\rfloor\right), & \text{otherwise}.
\end{cases}
$$

Update the answer with $best$.

## Problem 3

Finding
$$
\max \prod_{i=1}^{n}\Bigl(1 - (1 - r_i)^{m_i}\Bigr)
$$
is the same as finding
$$
\max \sum_{i=1}^{n}\log\Bigl(1 - (1 - r_i)^{m_i}\Bigr).
$$

Let
$$
f(i, j) = \log\Bigl(1 - (1 - r_i)^j\Bigr),
$$
which means the term contributed by stage $i$ when buying $j$ machines to produce $i$.

Thus we can solve the problem using DP.

First, compute all $f(i, j)$. Since the total number of terms is no more than
$$
n \cdot \left(\frac{B}{\min(c_i)}\right),
$$
this step takes
$$
O\!\left(n \cdot \frac{B}{\min(c_i)}\right)
$$
time.

Next, define $dp[i][b]$ as the maximum value of the sum of $\log(\cdot)$ terms for the first $i$ stages using total cost $b$.  
The recurrence is:
$$
dp[i][b] = \max\Bigl(dp[i-1][b - j c_i] + f(i, j)\Bigr).
$$

The time cost of this part is:
$$
\sum_{i=1}^{n}\sum_{b=0}^{B} O\!\left(\frac{b}{c_i}\right)
= O\!\left(\frac{nB^2}{\min(c_i)}\right).
$$

Thus, the total time complexity is:
$$
O\!\left(\frac{nB^2}{\min(c_i)}\right).
$$

## Problem 4
### (a)
Let item $i$ have weight $w_i$ and value $v_i$. We want to find

$$
\max \sum_{i \in S} v_i
\quad \text{s.t.} \quad
\sum_{i \in S} w_i \le W,
$$

where each item can be chosen at most once.

Define $dp[i][w]$ as the maximum total value using the first $i$ items with knapsack capacity $w$.  
Initialize $dp[0][w] = 0$ for all $0 \le w \le W$.

For item $i$, we either do not take it, or take it (if possible). The recurrence is:

$$
dp[i][w] =
\begin{cases}
dp[i-1][w], & w < w_i,\\[6pt]
\max\bigl(dp[i-1][w],\; dp[i-1][w-w_i] + v_i\bigr), & w \ge w_i.
\end{cases}
$$

There are $n(W+1)$ states and each transition takes $O(1)$, so the total time complexity is:

$$
O(nW).
$$

### (b)
The algorithm does not run in polynomial time. To describe the problem $P(n, W)$, we need $O(n + \log_2 W)$ bits, but the runtime is $O(nW)$. In some cases,

$$
O(nW) = 2^{O(n + \log_2 W)},
$$

thus the algorithm does not run in polynomial time.
## Problem 5

Assume there exist polynomial-time algorithms $a_1$ and $a_2$ that decide languages $L_1$ and $L_2$.

### Union
For $x \in L_1 \cup L_2$, run $a_1(x)$ and $a_2(x)$.  
If either one accepts, then accept; otherwise reject.  
The total running time is polynomial.

### Intersection
For $x \in L_1 \cap L_2$, run $a_1(x)$ and $a_2(x)$.  
If both accept, then accept; otherwise reject.  
The total running time is polynomial.

### Complement
For $x \in \overline{L_1}$, run $a_1(x)$.  
If $a_1(x)$ accepts, then reject; otherwise accept.  
The running time is polynomial.

### Concatenation
For $x \in L_1 L_2$, try all possible splits $x = yz$.  
For each split, run $a_1(y)$ and $a_2(z)$.  
If there exists a split such that both accept, then accept; otherwise reject.
There are at most $|x| + 1$ splits, so the total running time is polynomial.


### Kleene Star
For $x \in L_1^*$, check whether $x$ can be split into substrings
$$
x = y_1 y_2 \cdots y_k
$$
such that each $y_i \in L_1$.
This can be done in polynomial time.  
Thus the total running time is polynomial.

## Problem 6

### (a)
For any $L \in P$, we have $\overline{L} \in P$.  
Since $P \subseteq NP$, this implies $\overline{L} \in NP$, i.e., $L \in coNP$.  
Therefore, $P \subseteq coNP$.
### (b)
Assume that $P = NP$.  
Then $NP \subseteq coNP$.

For a language $L$, if $\overline{L} \in coNP$, then $L \in coNP$.  
Thus we have $NP = coNP$, which leads to a contradiction.

Therefore, $P \ne NP$.

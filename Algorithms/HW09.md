## Problem 1

### (a)

The worst case time complexity is $\Theta(n^2)$.

**Upper bound ($O(n^2)$):** For each **FIND** operation, the worst case occurs when we traverse from a leaf node to the root, visiting each node along the path. In the worst case, this path can contain up to $n$ nodes, resulting in a cost of $O(n)$ per **FIND** operation. Since there are $n$ nodes in total, performing **FIND** operations on all nodes yields a total cost of $O(n^2)$.

**Lower bound ($\Omega(n^2)$):** We can construct a worst case scenario where all $n$ elements form a single chain (a degenerate tree). In this configuration, performing **FIND** operations on all elements requires traversing paths of lengths $1, 2, \ldots, n$ respectively, resulting in a total cost of $1 + 2 + \cdots + n = \frac{n(n+1)}{2} = \Theta(n^2)$. This demonstrates that the worst case cost is $\Omega(n^2)$.

Therefore, the worst case time complexity is $\Theta(n^2)$. 

### (b)

Each tree edge can be visited at most twice during all **FIND** operations.

The first visit occurs when the edge is traversed during a **FIND** operation, as we follow the path from a node to the root. After this traversal, path compression redirects the edge to point directly to the root. The second visit occurs when we traverse this compressed edge in a subsequent **FIND** operation to reach the root.

Since each node (except the root) has exactly one edge above it, and there are $n-c$ non-root nodes (where $c$ is the number of trees), the total number of edge visits across all **FIND** operations is at most $2(n-c)$. Therefore, the total time complexity of all **FIND** operations is $O(n)$.

We use a segment tree to maintain the array $X[1, \ldots, n]$. Each node in the segment tree stores:
- The start and end indices of the segment it represents
- The number of 1s in that segment

For each node, its two children represent the left and right halves of the segment. The tree has $O(\log n)$ levels.

**LOOKUP operation:** Directly access the element $X[i]$ from the array, which takes $O(1)$ time.

**BLACKEN operation:** 
1. Find the leaf node corresponding to index $i$ (where $\text{start} = \text{end} = i$)
2. If $X[i]$ is currently 0, update it to 1 and increment the count of 1s in that node
3. Traverse upward to the root, incrementing the count of 1s for each node along the path

This operation takes $O(h) = O(\log n)$ time, where $h$ is the height of the tree.

**NEXTWHITE operation:**
Query the region from index $i$ to $n$ starting from the root. Traverse down the tree to find the first index $j \geq i$ such that $X[j] = 0$. This is done by checking if the left child contains any 0s (i.e., the number of 1s is less than the segment length), and recursively searching in the appropriate subtree. This operation takes $O(h) = O(\log n)$ time.

Each set represents a contiguous block of 1s, with the representative element being the rightmost (largest index) element in the block.

**LOOKUP operation:** Directly access the element $X[i]$ from the array, which takes $O(1)$ time.

**BLACKEN operation:**
1. When setting $X[i] \leftarrow 1$, we need to merge the set containing $i$ with adjacent sets if they exist
2. Use union by size: when merging two sets, always make the larger set's representative the new representative
3. Additionally, ensure the representative element is always the rightmost element in the merged set
4. Merge the smaller set into the larger set by updating pointers

**Time complexity analysis:** Each time an element is "moved" (i.e., its set is merged into a larger set), the size of the set containing that element at least doubles. Therefore, each element can be moved at most $O(\log n)$ times. Since there are $n$ elements, the total cost over all **BLACKEN** operations is $O(n \log n)$. With $n$ **BLACKEN** operations, the amortized time per operation is $O(\log n)$.

Return the representative element of the set that contains index $i$. Since the representative is the rightmost element in a contiguous block of 1s, it points to the next index with value 0 (or the end of the array). This operation takes $O(1)$ time with path compression.

## Bonus Problem

We use two BFS traversals to find the diameter of a tree.

**Algorithm:**

1. **First BFS:** Start from an arbitrary vertex $r$ (root). Perform BFS and find a vertex $v_1$ that is at maximum distance from $r$. The vertex $v_1$ is the last vertex visited in this BFS.

2. **Second BFS:** Start from vertex $v_1$. Perform BFS and find a vertex $v_2$ that is at maximum distance from $v_1$. The vertex $v_2$ is the last vertex visited in this BFS.

3. **Result:** The distance between $v_1$ and $v_2$ is the diameter of the tree.

**Time complexity:** 
- First BFS: $O(|V| + |E|) = O(|V|)$ (since a tree has $|E| = |V| - 1$)
- Second BFS: $O(|V| + |E|) = O(|V|)$
- **Total: $O(|V|)$**

### (a)

The claim is true.

Consider a directed graph $G = (V, E)$ with three vertices $V = \{a, b, c\}$ and edges $E = \{(a, b), (b, c)\}$, forming a chain $a \to b \to c$. The vertex $b$ has both an incoming edge (from $a$) and an outgoing edge (to $c$).

Now consider the following DFS execution:
1. Start DFS from vertex $c$: $c$ is explored and becomes a tree by itself.
2. Start DFS from vertex $b$: Since $c$ has already been visited, $b$ is explored but cannot reach $c$. Thus $b$ becomes a separate tree in the DFS forest.

In the resulting DFS forest, $b$ is a tree that contains only itself, and $b$ has both incoming and outgoing edges in the original graph $G$. This proves the claim. 

### (b)

Such an algorithm exists.

For an undirected graph $G = (V, E)$, if $|E| \geq |V|$, then $G$ must contain a cycle. This is because a tree has exactly $|V| - 1$ edges.

**Algorithm:**
1. Count edges while traversing the adjacency list. If we encounter $|V|$ or more edges, immediately return `true` (cycle exists). This step takes at most $O(|V|)$ time since we stop once we've seen $|V|$ edges.
2. If the procedure didn't return in step 1, then $|E| < |V|$, which means $|E| = O(|V|)$. Perform DFS on the graph:
   - If we encounter a back edge (an edge to a previously visited vertex that is not the parent), return `true`
   - If DFS completes without finding a back edge, return `false`

**Time complexity:** 
- Step 1: $O(|V|)$ time (we stop after counting $|V|$ edges)
- Step 2: $O(|V| + |E|) = O(|V|)$ time (since $|E| < |V|$ when we reach this step)

Therefore, the overall time complexity is $O(|V|)$, independent of $|E|$. 

### (c)

We implement Kahn's algorithm for topological sorting using a queue to efficiently find vertices with in-degree 0.

**Algorithm:**

**Step 1: Initialize in-degree counts** ($O(|V| + |E|)$ time)
- Create an array `inDegree` of size $|V|$, initialized to 0
- Iterate through the adjacency list representation:
  - For each edge $(u, v)$ in the graph, increment `inDegree[v]`
- This step takes $O(|V| + |E|)$ time since we visit each vertex and each edge once

**Step 2: Initialize queue with zero in-degree vertices** ($O(|V|)$ time)
- Scan through all vertices and add those with `inDegree[v] = 0` to a queue
- This takes $O(|V|)$ time

**Step 3: Process vertices** ($O(|V| + |E|)$ time)
- While the queue is not empty:
  1. Dequeue a vertex $u$ and output it (add to topological order)
  2. For each neighbor $v$ of $u$ (i.e., for each edge $(u, v)$):
     - Decrement `inDegree[v]`
     - If `inDegree[v]` becomes 0, enqueue $v$
- Each vertex is dequeued exactly once: $O(|V|)$
- Each edge is processed exactly once: $O(|E|)$

**Time complexity:** 
- Step 1: $O(|V| + |E|)$
- Step 2: $O(|V|)$
- Step 3: $O(|V| + |E|)$
- **Total: $O(|V| + |E|)$** 


## Problem I.1

We need to find a vertex $v$ with in-degree $|V|-1$ and out-degree $0$. This means all other vertices point to $v$, and $v$ points to no other vertex.

**Step 1:** Find a candidate vertex $v$ in $O(|V|)$ time.

```python
v = 1
for j = 2 to n:
    if G[v][j] == 1:
        v = j
```

The key observation is that if a vertex has an outgoing edge, it cannot be the vertex we're looking for (since we need out-degree $0$). This step finds a vertex $v$ that has no outgoing edges to any vertex with index greater than $v$.

**Step 2:** Verify that $v$ is indeed the desired vertex in $O(|V|)$ time.

Check the $v$-th column of the adjacency matrix (i.e., all incoming edges to $v$). If all elements in column $v$ are $1$ (except possibly $G[v][v]$ which may be $0$ or $1$ depending on self-loops), then $v$ has in-degree $|V|-1$ and we return `true`. Otherwise, return `false`.

```python
for i = 1 to n:
    if i != v and G[i][v] == 0:
        return false
return true
```

**Time complexity:** Both steps take $O(|V|)$ time, so the overall time complexity is $O(|V|)$. 

## Problem I.2

This is a graph coloring problem. Construct an undirected graph $G = (V, E)$ where:
- Each vertex represents a movie $M_1, M_2, \ldots, M_k$
- An edge exists between two movies if there is a customer who wants to see both movies

Since each customer wants to see two movies, and movies are shown on Saturday and Sunday, the two movies desired by the same customer must be shown on different days. Therefore, we need to 2-color the graph such that adjacent vertices (movies desired by the same customer) have different colors.

**Algorithm:**
1. Build the graph as described above
2. Use BFS to perform 2-coloring:
   - Start from an arbitrary vertex, color it as "Saturday" (color 0)
   - For each vertex, color all its adjacent vertices with the opposite color
   - If a conflict occurs (an adjacent vertex already has the same color), return "impossible"
   - Otherwise, the coloring gives a valid schedule

**Time complexity:** $O(v + e)$ where $v$ is the number of movies and $e$ is the number of customer preferences. 

## Problem I.3

We use a postorder DFS traversal with memoization to evaluate the DAG. The key idea is to compute each subexpression only once and cache the result.

**Algorithm:**
1. Perform a postorder DFS starting from the root of the DAG
2. For each node visited:
   - If the node is a leaf (contains an integer), return its value
   - If the node is an internal node (contains an operator):
     - Recursively evaluate its left and right children
     - Apply the operator to the two computed values
     - Store the result in the node (memoization) and return it
3. If a node has already been evaluated (its result is cached), return the cached value directly

**Time complexity:** 
- Since common subexpressions are removed in the DAG representation, each node represents a unique subexpression
- By using memoization, each node is evaluated at most once
- Each edge is traversed at most once during the DFS
- Each of the $n$ nodes is visited and evaluated at most once: $O(n)$
- Each of the $m$ edges is traversed at most once: $O(m)$
- Therefore, the total time complexity is $O(n + m)$

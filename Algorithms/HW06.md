## Problem 1
### Basic Idea
For each node, when the **print** function is called:

1. **Print** the node itself.  
2. **Recursively** call the print function on each of its children, printing all their keys in turn.

This ensures that every node and all of its descendants are printed in a **preorder traversal** manner — visiting the current node first, followed by all its children.

### Pseudocode
```python
PRINT(node root):
    if root == null:
        return
    print(root.key)
    PRINT(root.firstchild)
    PRINT(root.nextsibling)
```
## Problem 2
### (a)

We start by selecting a node. If this node is removed, at most three subtrees are generated. If the largest subtree contains more than $n / 2$ nodes, we move in that direction and repeat the process.

When we move to a new node, the two smaller subtrees from the previous step are merged, forming a combined subtree with at most $n / 2$ nodes. Meanwhile, the previously largest subtree is further divided, making it smaller than before.

As this process continues, in each iteration, as long as the largest subtree has more than $n / 2$ nodes, the subtree on the opposite side (the one we came from) becomes strictly smaller than $n / 2$. All other subtrees remain smaller than or equal to the previous largest one. Therefore, the largest subtree continually decreases in size.

Consequently, we are guaranteed to find a node that divides the tree into three subtrees, each containing no more than $n / 2$ nodes.

### (b)
#### Basic Idea
First, recursively scan through the tree to calculate the **size of each subtree** for every node.

After obtaining the subtree sizes, apply the **algorithm described in (a)** to identify the **central vertex** — the node that divides the tree into subtrees, each containing no more than half of the total nodes.

#### Pseudocode
```python
CALCULATESIZE(node n)
    if n == null:
        return 0
    leftsize = CALCULATESIZE(n.leftchild)
    rightsize = CALCULATESIZE(n.rightchild)
    node.leftsize = leftsize
    node.rightsize = rightsize
    return leftsize + rightsize + 1
FINDCENTRAL(node n, ts):# tree size
CALCULATESIZE(n)
    while true:
        if n.leftsize >  ts / 2:
            n = n.leftchild
            continue
        if n.rightsize > ts / 2:
            n = n.rightchild
            continue
        return n
```
#### analyze
In the **CALCULATESIZE** stage, each node is checked exactly once, resulting in a time complexity of $O(n)$.

Inside the `while` loop, the number of nodes checked is less than $n$, so the time cost for this part is also $O(n)$.

Therefore, the **total time complexity** of the algorithm is:

$$
O(n) + O(n) = O(n)
$$

We observe that two different binary trees — one with **A** as the root and **B** as the left child, and the other with **A** as the root and **B** as the right child — both share the same preorder traversal **"A B"** and the same postorder traversal **"B A"**.

This means that the reconstruction of a binary tree from preorder and postorder traversals is **not guaranteed to be unique**.  
Therefore, an algorithm to uniquely reconstruct the tree **does not exist**.

### (c)
#### Basic Idea
First, if both the preorder and postorder sequences consist of only one element, the tree is simply that single node.

If not, the sequences must contain at least three elements. In the **preorder** sequence, `pre[0]` represents the root, and `pre[1]` represents the root of the left subtree. In the **postorder** sequence, `post[-1]` is the root, and `post[-2]` is the root of the right subtree.

By removing `pre[0]` (the root) from the preorder sequence, the elements `pre[1]` and `post[1]` can be used to divide both sequences into two parts, corresponding to the two subtrees.

The reconstruction can then proceed **recursively** on these two subtrees.

#### Pseudocode
```python
MAKETREE(pre, post):
    root = node(pre[0])
    if pre.len == 1:
        return root
    root.leftchild = MAKETREE(pre[1 to pre.index(post[-2]) - 1], post[0 to post.index(pre[1])])
    root.rightchild = MAKETREE(pre[pre.index(post[-2]) to -1], post[post.index(pre[1]) + 1 to -2])
    return root
```
## Bonus Problem
Use a Morris Traversal mathod. 
### Basic Idea
For the current node `cur`:

- If no **left child** exists, **traverse** the node and move to its **right child**.
- Otherwise, find the **largest element** in its left subtree, denoted as `pre`.

Then, proceed as follows:

1. If the **right child** of `pre` is `null`, set `pre->rightchild = cur` and move `cur` to its left child.
2. If the **right child** of `pre` is `cur`, reset `pre->rightchild` to `null`, **output** `cur`, and move `cur` to its right child.

Since each node is visited exactly once and no additional data structures are required, the **time complexity** is $O(n)$ and the **space complexity** is $O(1)$.

### Pseudocode
```python
MORRISTRAVERSAL(root, vist):# vist is hight order function
    cur = root
    while(cur != null):
        if(cur.leftchild == null):
            vist(cur)
            cur = cur.rightchild
        else:
            pre = cur.leftchild
            while(pre.rightchild != null and pre.rightchild != cur):
                pre = pre.rightchild
            if pre.rightchild == null:
                pre.rightchild = cur
                cur = cur.leftchild
            else:
                pre.rightchild = null
                vist(cur)
                cur = cur.rightchild
```

## Problem 5
### Basic Idea
Given two binary search trees **T₁** and **T₂**, where every key in **T₁** is smaller than any key in **T₂**, we can merge them into a single BST as follows:

We iteratively move to the **left child** of the current node in **T₂** until we reach the **smallest node** in **T₂**. This traversal takes $O(h)$ time, where $h$ is the height of the tree.

Once the smallest node in **T₂** is found, we assign the **root of T₁** as the **left child** of this smallest node.  
As a result, we obtain a **connected binary search tree** that maintains the BST property.

### Pseudocode
```python
CONCATENATE(t1, t2):
    t = t2
    while t2.leftchild != null:
        t2 = t2.leftchild
    t2.leftchild = t1
    return t
```

## Problem 2
### (a)
#### Basic Idea
First use a DFS to find a topological order of all vertices; this takes $O(V + E)$.  
Then, following the topological order, visit each vertex and add `cnt` to its adjacent vertices.  
Since each vertex and edge is processed once, this step also takes $O(V + E)$.

Therefore, the total time complexity is:

$$
O(V + E).
$$

#### Pseudocode
```python
FindPath(graph G):
    topo-sort => Gt
    for v in Gt:
        v.cnt = 0
    s.cnt = 1
    for v in Gt:
        for (v, u) in G:
            u.cnt += v.cnt
    return t.cnt
```
### (b)
#### Basic Idea
First use a DFS to find a topological order of all vertices; this takes $O(V + E)$.  
Then, following the topological order, visit all vertices and iterate through each edge, deciding whether the edge can optimize the start time of its end vertex.  
This step also takes $O(V + E)$.

Therefore, the total time complexity is:

$$
O(V + E).
$$

#### Pseudocode
```python
FindStart(graph G):
    topo-sort => Gt
    for v in Gt:
        v.start = 0
    for v in Gt:
        for (v, u) in G:
            if v.start + v.weight > u.start:
                u.start = v.start + v.weight
    return t.start
```
### (c)
#### Basic Idea
First perform the procedure in (b) to find the latest start time of $t$.  
Then, in reversed topological order, visit all vertices and iterate through each edge that ends at the current vertex, deciding whether the edge can optimize the latest start time of its start vertex.

The total time complexity is:

$$
O(V + E).
$$

#### Pseudocode
```python
FindLate(graph G):
    t.late = FindStart(G) # in (b)
    reverse-topo-sort => Gtr
    for v in Gtr:
        v.late = t.late
    for v in Gtr:
        for (u, v) in G:
            if u.late > v.late - u.weight:
                u.late = v.late - u.weight
```

## Problem 3
### Basic Idea
Produce a new matrix with  
$S[i, j] = -\log(R[i, j])$,  
which takes $O(n^2)$ time.

Then run the Floyd–Warshall algorithm on the new matrix $S$,  
which takes $O(n^3)$ time.

Finally, find the smallest value $v$ in $S$ and convert it by:

$$
2^{-v}.
$$

The total runtime is:

$$
O(n^3).
$$
### Pseudocode
```python
FindLargest(matrix R):
    matrix S
    for i from 1 to n:
        for j from 1 to n:
            S[i, j] = -log(R[i, j])
            dist[i, j, 0] = S[i, j]
    for r from 1 to n:
        for i from 1 to n:
            for j from 1 to n:
                dist[i, j, r] = dist[i, j, r - 1]
                if dist[u, v, r] > dist[u, r, r - 1] + dist[r, v, r - 1]:
                    dist[u, v, r] = dist[u, r, r - 1] + dist[r, v, r - 1]
    return 2^(-max(dist))
```

## Problem 4

Use a DP method. Let $dp[u][t]$ denote the shortest path from $v$ to $u$ using $t$ steps.

The transition is:
$$
dp[u][t] = \min\bigl(dp[a][t-1] + \text{weight}(a, u)\bigr),
$$
where $a$ ranges over all vertices that have an edge pointing to $u$.

Compute in increasing order of $t$ from $0$ to $k$. Initialize all $dp$ values to $\inf$, and set:
$$
dp[v][0] = 0.
$$
Then compute all transitions for $t = 1, 2, \dots, k$.

Finally, $dp[w][k]$ is the target value.

For each layer, the time complexity is $O(E)$, so the total time complexity is:
$$
O(Ek).
$$

## Problem 5
### (a)
This problem satisfies optimal substructure. We have:

$$
OPT(S) = \max(a_n + SOL(S')).
$$

Assume for the sake of contradiction that $OPT(S)$ is the optimal solution of $S$, but $SOL(S')$ is not the optimal solution of the subproblem.  
Then there must exist a solution for $S'$ such that:

$$
OPT(S') > SOL(S').
$$

Using $OPT(S')$, we can combine it with $a_n$ to obtain a solution better than $OPT(S)$, which is a contradiction.  
Thus, $OPT(S)$ must be formed by optimal solutions to its subproblems.

### (b)
If the problem is defined with only one parameter $len$, which represents the length of the remaining rod, then the problem does not satisfy optimal substructure.

We can use a counterexample to justify this.  
Let the total length be $4$, with prices $(2, 5, 10)$ and remaining allowed quantities $(1, 114514, 114514)$.  
The optimal solution to the subproblem is cutting the rod as $2 + 2$, which is **not** a substructure of the optimal solution to the entire problem.


However, if we define the problem with two parameters, $len$ and $v$, where $v$ is a vector that records the remaining number of pieces of length $i$ that are allowed to be produced, then it satisfies optimal substructure. The justification is the same as in (a).

## Problem 6

Use a DP-based algorithm.

In this problem, there is a total of $n$ numbers. Maintain two tables:  
$max[i][j]$ is the maximum value, and $min[i][j]$ is the minimum value that can be calculated using the numbers between the $i$-th and the $j$-th positions.  
Let $op[k] \in \{+,-\}$ be the sign right before the $k$-th number (i.e., between the $(k-1)$-th and $k$-th numbers).

For $k \in [i+1, j]$, we update:

$$
\begin{aligned}
&\text{if } op[k] = +: \\
&\quad max[i][j] = \max\bigl(max[i][j],\; max[i][k-1] + max[k][j]\bigr),\\
&\quad min[i][j] = \min\bigl(min[i][j],\; min[i][k-1] + min[k][j]\bigr);\\[4pt]
&\text{if } op[k] = -: \\
&\quad max[i][j] = \max\bigl(max[i][j],\; max[i][k-1] - min[k][j]\bigr),\\
&\quad min[i][j] = \min\bigl(min[i][j],\; min[i][k-1] - max[k][j]\bigr).
\end{aligned}
$$

Using this recurrence, we can determine an evaluation order to solve all DP states.  
Each grid in the DP matrix depends on grids that are above and to the right, so we fill from the diagonal toward the left-down corner. At:

$$
max[1][n]
$$

we obtain the final result.

For each grid, the time cost is $O(n)$, thus the total time complexity is:

$$
O(n^3).
$$

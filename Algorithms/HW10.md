### (b)
If $x$ and $\neg x$ are in the same SCC, there must exist a path:

$$
x \rightarrow m_1 \rightarrow m_2 \rightarrow \cdots \rightarrow m_n \rightarrow \neg x.
$$

By implication graph construction, this induces the reverse implications:

$$
\neg m_1 \rightarrow \neg x,\quad
\neg m_2 \rightarrow \neg x,\quad
\cdots,\quad
\neg m_n \rightarrow x.
$$

Thus, the corresponding clauses include:

$$
(x \lor \neg m_n),\quad
(x \lor m_1),\quad
(\neg x \lor \neg m_1),\quad
(\neg x \lor m_n).
$$

Together these imply:

$$
x \lor (\neg m_1 \land m_n \land m_1 \land m_n),
$$

which is guaranteed to be false.

Therefore, the formula has **no satisfying assignment**.

### (c)
We provide the following algorithm:

Compute all SCCs of $G$ using either **Kosaraju** or **Tarjan** algorithm.  
Then perform a **topological sort** on the SCC graph.  
For each pair $(x, \neg x)$, assign:

- the one whose SCC appears **later** in the topological order → **TRUE**,  
- the other one → **FALSE**.

Apply this rule to every variable $x$.

**Proof of correctness**

The assignment is correct **iff** no edge of the form **TRUE → FALSE** exists.

Assume for the sake of contradiction that such an edge exists:
$$
x \rightarrow y.
$$

Since $x$ is assigned **TRUE**, its negation $\neg x$ must lie in an SCC that appears **before** the SCC of $x$ in the topological order.  
Similarly, since $y$ is assigned **FALSE**, its negation $\neg y$ must lie in an SCC that appears **after** the SCC of $y$.

Thus the SCC of $\neg x$ occurs **before** that of $x$,  
and the SCC of $\neg y$ occurs **after** that of $y$.  
Therefore, $\neg x$ and $\neg y$ cannot lie in the same SCC nor in adjacent SCCs.

Consequently, the reverse implication  
$$
\neg y \rightarrow \neg x
$$
cannot exist in $G$, contradicting the fact that implication graphs enforce the rule:
$$
(x \rightarrow y) \implies (\neg y \rightarrow \neg x).
$$

Hence, no TRUE → FALSE edge can exist, and the algorithm is correct.

### (d)
Using the algorithm in (c):

- computing SCCs takes $O(V + E)$,
- performing a topological sort takes $O(V + E)$,
- assigning TRUE/FALSE also takes $O(V + E)$.

Therefore, the total running time is:

$$
O(V + E).
$$

## Problem 4
### (a)
Since all edges have distinct weights, for every cut of $G$ there is a unique light edge crossing that cut.  
From Problem 3(b), we know that if every cut has a unique light edge, then the minimum spanning tree (MST) is unique.  
Therefore, $G$ has a unique MST.
For the specific graph:

- $(a, b) = 1$  
- $(a, c) = 2$  
- $(a, d) = 5$  
- $(b, c) = 4$  
- $(b, d) = 7$  
- $(c, d) = 3$

The MST is unique, as expected.  
However, the **second** MST can be:

- $1 + 3 + 4$ (using edges $(a,b)$, $(c,d)$, $(b,c)$), or  
- $1 + 2 + 5$ (using edges $(a,b)$, $(a,c)$, $(a,d)$).

Since both have the same total weight, the second MST is **not** necessarily unique.

### (b)
Assume for the sake of contradiction that obtaining the second minimum spanning tree (SMST) from the MST requires two or more edge exchanges.

Let the set of exchanged edges be grouped into pairs  
$(a_1, b_1), (a_2, b_2), \dots, (a_n, b_n)$  
where each $a_i$ is an edge in the MST and $b_i$ is an edge crossing the same cut as $a_i$.

Since the MST contains the lightest edge across every cut, we have  
$$a_i < b_i \quad \text{for all } i.$$

Therefore, performing **only** the exchange $(a_1, b_1)$ produces a spanning tree whose weight is:

- greater than the weight of the MST, and  
- smaller than the weight of the supposed SMST formed by multiple exchanges.

This contradicts the assumption that multiple exchanges were necessary to reach the SMST.

Thus, **exactly one** edge must differ between the MST and the second minimum spanning tree.

#### Basic Idea
Perform a DFS on $G$.  
For each vertex $v$, run another DFS starting from $v$ to compute the maximum‐weight edge on the path between $v$ and every other reachable vertex.

The inner DFS is executed $O(V)$ times, and each DFS costs $O(V)$ time.  
Therefore, the total time complexity is:

$$O(V) \times O(V) = O(V^2).$$

### (d)
First compute the MST of $G$, which takes $O(V + E) = O(V^2)$ time for a dense graph.  
Then, for every pair of vertices $(x, y)$ in the MST, check whether the edge $(x, y)$ exists in the original graph.  

If it does, compute the weight increase obtained by replacing the maximum edge on the MST path between $x$ and $y$, denoted `MAX[x][y]`, with the edge $(x, y)$.

Among all such possible replacements, choose the one with the smallest weight increase and perform this exchange.  
The resulting spanning tree is the **second minimum spanning tree**.

## Problem 5
### Basic Idea
Use a greedy strategy:  
At each step, among all intervals whose left endpoint is $\le \text{currentEnd}$, choose the interval that extends **as far right as possible**.  
Add this interval to the solution and update `currentEnd` to its right endpoint.  
Repeat until the whole target interval $Y$ is covered.

To implement this efficiently, we first sort all intervals by their left endpoints.
### Pseudocode
```python
SMALLEST(L, R):
    L, R -> I[(L[1], R[1]), ..., (L[n], R[n])]
    sort(I) # by L, time cost is O(nlogn)
    currentend = I[0].L
    Y = []
    while !I.empty:
        largest = I[0]
        while I[0].L <= currentend and !I.empty:
            if I[0].R > largest.R:
                largest = I[0]
            I.popfront()
        if largest.R > currentend:
            Y.pushback(largest)
            currentend = largest.R
    return Y.size
```
### Time Complexity
The most expensive operation is sorting, which takes **$O(n \log n)$** time.  
The subsequent scanning/selection process takes **$O(n)$** time, because each interval is examined (or popped) at most once.

Thus, the total time complexity is:

$$
O(n \log n) + O(n) = O(n \log n).
$$
### Proof of Correctness

#### Greedy Choice Property
Suppose during some step we select the interval  
$$
i = [L, R],
$$
where $L \le \text{currentEnd}$ and $i$ has the **largest** right endpoint $R$ among all such intervals.

Assume, for contradiction, that $i$ does **not** appear in an optimal solution.  
Then the optimal solution must contain some interval  
$$
j = [L_j, R_j],
$$
with $L_j \le \text{currentEnd}$ but  
$$
R_j < R.
$$

Since $i$ covers everything that $j$ covers and more, replacing $j$ with $i$ will not worsen the solution.  
This contradicts the assumption that the optimal solution does not use $i$.  
Thus, the greedy choice is always safe.
#### Optimal Substructure
Let the first chosen interval in the optimal solution be $s$, size of $s$ is one.  
After selecting $s$, the remaining uncovered part forms the subproblem $A_c$.

If the $SOL(A_c)$, were not optimal, meaning

$$
SOL(A_c) > OPT(A_c),
$$

then replacing the  $SOL(A_c)$ with $OPT(A_c)$ yield a better global solution:

$$
OPT(A_c) + 1 < SOL(A_c) + 1 = OPT(A),
$$

contradicting the optimality of the original solution.

Thus, the problem has optimal substructure.

## Problem 1

### Algorithm
Use a greedy strategy. First divide $n$ by $2$ until $n$ becomes an odd number;  
then decrease $n$ by $1$ and continue dividing by $2$.  
Repeat this process until $n = 1$.  
The reverse sequence of these operations gives the target sequence.

The number of decrement operations is no more than the number of division operations,  
and each division by $2$ reduces $n$ by at least half.  
Thus there are at most $\log_2 n$ such operations,  
so the total time complexity is:

$$
O(\log n).
$$

### Proof
Define $\text{OPT}(n)$ as the minimum number of operations required to obtain $n$ starting from $1$.  
It is clear that if $a < b$, then $\text{OPT}(a) \le \text{OPT}(b)$.

When $n \ge 2$, we have:

$$
\frac{n}{2} \le n - 1,
$$

which implies

$$
\text{OPT}\!\left(\frac{n}{2}\right) \le \text{OPT}(n - 1).
$$

The number of operations from $\frac{n}{2}$ to $n$ and from $n - 1$ to $n$ is the same.  
Thus, cutting the target value by half is always at least as good as decreasing by $1$,  
which justifies the correctness of the greedy strategy.

## Problem 4
### Basic Idea
Use DFS to visit each edge and change the weight $w$ of each edge to $-\log(w)$.  
Then apply Dijkstra's algorithm to find the shortest path on the modified graph.

Since Dijkstra runs in $O(E \log V)$ time and DFS runs in $O(E + V)$ time,  
the total time complexity is:

$$
O(E \log V + V).
$$

### Pseudocode
```python
DFS(G, s):
    s.visited = true
    for (s, v) in G:
        if(s, v).visited = false:
            (s, v).weight = -log(s, v).weight
            (s, v).visited = true
        if not v.visited:
            DFS(G, v)

Dijkstra(G, s):
    for each u in G:
        u.dis = INF, u.parent = null
    s.dist = 0
    Build priority queue Q based on dist
    while not Q.empty():
        u = Q.extractMin()
        for each edge (u, v) in G:
            if v.dist > u.dist + w(u, v):
                v.dist = u.dist + w(u, v)
                v.parent = u
                Q.updateKey(v)

FindPath(G, s, t):
    DFS(G, s)
    Dijkstra(G, s)
```

## Problem 5
### Basic Idea
Use a Dijkstra-like method, but instead of computing the minimum distance to a node,  
compute the **minimum required height** to reach each node.

The time complexity is the same as Dijkstra's algorithm, which is:

$$
O(E \log V).
$$

### Pseudocode
```python
SpecialDijkstra(G, s):
    for each u in G:
        u.height = 0, u.parent = null
    s.height = INF
    Build priority queue Q based on height
    while not Q.empty():
        u = Q.extracMax()
        for each edge (u, v) in G:
            if min(u.height, (u, v).height) > v.height:
                v.height = min(u.height, (u, v).height)
                v.parent = u
                Q.updateKey(v)
```

Since each edge is visited once per pass and there are $O(n)$ passes,  
the total time complexity is:

$$
O(VE).
$$

For each vertex, if its nearest vertex is $x$ steps away,  
then after $x$ passes the algorithm finds the correct solution.  
Thus, after $n - 1$ passes, all vertices find their nearest vertex.

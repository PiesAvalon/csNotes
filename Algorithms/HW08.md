## Problem 1
### (a)

We define 
$$
K_{i,j} =
\begin{cases}
1, & \text{if the $i$th and $j$th elements are in the same slot} \\
0, & \text{otherwise}
\end{cases}
$$


For each $i, j$,  
$$
E(K_{i,j}) = P(K_{i,j} = 1) = \frac{1}{m}.
$$

Then the expected number of collisions is

$$
E(\#\text{collisions}) 
= E\left(\sum_{1 \le i < j \le n} K_{i,j}\right)
= \sum_{1 \le i < j \le n} E(K_{i,j})
= \sum_{1 \le i < j \le n} \frac{1}{m}
= \binom{n}{2} \cdot \frac{1}{m}
= \frac{n(n-1)}{2m}.
$$

### (b)

We pick a slot for each element in order.  
For the $i$th item, the probability of **not** having a collision is $\dfrac{m - i + 1}{m}$.  

Thus, the probability of having a **perfect hash** (no collisions) is:

$$
P = \prod_{i=1}^{n} \frac{m - i + 1}{m}
= \frac{m!}{m^n (m - n)!}.
$$

### (c)

Let $X$ be the number of different random hash functions we test.  

$$
X \sim \text{Geom}\!\left(\frac{m!}{m^n (m - n)!}\right).
$$

The expected number of tries is:

$$
E(X) = \frac{1}{P} = \frac{m^n (m - n)!}{m!}.
$$

### (d)
The probability **one function is not perfect**: $1 - P$.
Assuming picks is independent, the probability none of $N$ functions is perfect:
$$
(1 - P)^N \;=\; \left(1 - \frac{m!}{(m-n)! \, m^n}\right)^{N}.
$$

### (e)
We want the smallest integer $k$ such that  

$$
1 - (1-P)^k \ge 1 - \frac{1}{n} .
$$
Solving this, we have
$$
k_{\min}
= \left\lceil \frac{\ln n}{-\ln(1-P)} \right\rceil
= \left\lceil \frac{\ln n}{-\ln\!\left(1-\frac{m!}{(m-n)!\,m^n}\right)} \right\rceil .
$$

## Problem 2
### Basic Idea

We use the following data structures:
- An **array** `A` to store all bits.  
- A **linked list** `L`, where each node stores the index of a bit that is currently set to one.  
- An **array of pointers** `P`, where `P[i]` points to the corresponding node in `L` if the $i$-th bit is one; otherwise, it is `null`.

**On `INCREMENT`:**
- Perform the necessary bit flips in `A`.  
- If a bit is flipped **from one to zero**, use `P[i]` to locate the corresponding node in `L`, delete it, and set `P[i] = null`.  
- If a bit is flipped **from zero to one**, add its index to `L`, and set `P[i]` to point to the newly created node in `L`.

**On `RESET`:**
- Iterate through `L`, setting all corresponding bits in `A` to zero.  
- Delete each node in `L` and set its pointer in `P` to `null`.

### Pseudocode
```python
class COUNTER:
    linkedList L # double linked list
    array A
    array P
    INCREMENT():
        i = 0
        while A[i] == 1:
            A[i] = 0
            L.delete(P[i])
            P[i] = null
            i++
        L.addFront(i)
        A[i] = 1
        P[i] = L.front()
RESET():
        cur = L.head
        while cur != null:
            A[cur.data] = 0
            P[cur.data] = null
            cur = cur.next
        L.head = null
        L.tail = null
```
### Analysis
Let  
- $\hat{c_i} = 2$ if the $i$th operation is **INCREMENT**,  
- $\hat{c_i} = 0$ if the operation is **RESET**.  

In each **INCREMENT** operation, one zero bit is converted to one.  
This bit flip costs 1 unit, so the “balance” accumulated during each INCREMENT  
is equal to the **number of new ones minus the number of new zeros**.  
Hence, the remaining balance corresponds to the **number of bits currently set to one**.

A **RESET** operation will consume all remaining balance (since all bits are reset to zero).

Therefore,
$$
\sum_{i=0}^{n} c_i \le \sum_{i=0}^{n} \hat{c_i} = O(n).
$$


## Problem 3

We use the **potential method**.  
Define the potential function as $\Phi(D_i) = c n$,  
where $c$ is a constant and $n$ is the current size of the queue.  
It is obvious that $\Phi(D_i) \ge \Phi(D_0)$ for all $i$.

The amortized cost is defined as  
$$
\hat{c_i} = c_i + \Phi(D_i) - \Phi(D_{i-1}).
$$

#### When the $i$ th operation is **PUSH**:
$$
\hat{c_i} = 1 + c (n + 1) - c n = O(1).
$$

#### When the $i$ th operation is **PULL**:
$$
\hat{c_i} = 1 + c (n -1) - c n = O(1).
$$

#### When the $i$ th operation is **DECIMATE**:

$$
\hat{c_i} = kn + cn - c\lceil 0.1 n \rceil - c n \le kn + c n - 0.1 c n - c n = (k - 0.1 c) n.
$$

If we choose $c > 10k$, then $\hat{c_i} < 0$.

## Problem 4
Now we only consider the time cost of push and pop. For each iteration of the `for` loop, set the amortized cost to
$$
\hat{c_i} = 2.
$$

Each **PUSH** or **POP** costs $1$. Since each element is **PUSH**ed exactly once and **POP**ped at most once, the total actual cost satisfies
$$
\sum_{i=1}^{n} c_i \;\le\; \sum_{i=1}^{n} \hat{c_i} \;=\; 2n \;=\; O(n).
$$

We consider the time cost of the `for` and `while` statements.

- The `for` loop executes $O(n)$ times.
- The `while` loop executes at most $2n = O(n)$ times.

Thus, the total time cost of these control statements is $O(n)$.

Overall, the total cost is
$$
O(n) + O(n) = O(n).
$$


## Problem I.1
### Basic Idea

We use a **vector** to store the items, and the vector is **unordered**.

- On **INSERT(S, x)**:  
  Append $x$ to the back of $S$.  
  This operation takes $O(1)$ time.

- On **DELETELARGERHALF(S)**:  
  Use **QuickSelect** with the **Median-of-Medians** strategy to find the median element of the vector — this takes $O(n)$ time.  
  Then, scan the vector and keep only the elements **less than the median** — this also takes $O(n)$ time.

### Pseudocode
```python
class WEIRDSET:
    vector v
    INSERT(x):
        v.pushback(x) # O(1)
    DELETELARGERHALF():
        mdian = quickselectMedian(v) # use median of median, takes O(n) time
        vector newv
        for i in v:
            if i <= mdian:
                newv.pushback(i)
        v = newv
    PRINT(): # O(|S|) time
        for i in v:
            print(i)
```
### Analysis

We assume that the vector handles the `push_back` operation in $O(1)$ time and handles iteration in $O(n)$ time.

We also assume that:
- The time cost of a single **INSERT** operation is less than $k_1$.
- The time cost of a single **DELETELARGERHALF** operation is less than $k_2 n$, where $n$ is the current size of the set.

Let the amortized cost be defined as  
$$
\hat{c_i} = k_1 + k_2.
$$

Intuitively, this means we account for the cost of deleting an item at the time of its insertion,  
so the remaining “balance” in the account is always non-negative.  
This can be easily proven by induction.

Thus, for $m$ total operations, the total time cost is:
$$
T = \sum_{i = 1}^{m} c_i \le \sum_{i = 1}^{m} \hat{c_i} = m (k_1 + k_2) = O(m).
$$


## Problem I.2
### Basic Idea
We use two stacks to simulate the **QUACK** structure:

- **Stack S1** stores the **left half**, with the **leftmost element on top**.  
- **Stack S2** stores the **right half**, with the **rightmost element on top**.

**On `QUACKPUSH(x)`**:  
Push $x$ into `S1`.

**On `QUACKPOP()`**:  
- If `S1` is **not empty**, pop from `S1`.  
- If `S1` is **empty**, use a temporary stack `S3` to move the **left half** of `S2` into `S1`,  
  then pop from `S1`.

**On `QUACKPULL()`**:  
- If `S2` is **not empty**, pop from `S2`.  
- If `S2` is **empty**, use `S3` to move the **right half** of `S1` into `S2`,  
  then pop from `S2`.

### Pseudocode
```python
class QUACK:
    stack S1, S2, S3
    QUACKPUSH(x):
        S1.push(x)
    QUACKPOP():
        if S1 is not empty:
            return S1.pop()
        pop half from S2, push them into S3
        pop remaining from S2, push them into S1
        pop all from S3, push them into S2
        return S1.pop()
    QUACKPULL():
        if S2 is not empty:
            return S2.pop()
        pop half from S1, push them into S3
        pop remaining from S1, push them into S2
        pop all from S3, push them into S1
        return S2.pop()
```
### Analysis
$$
\Phi(D_i) = c \, \big|\, |S_1| - |S_2| \,\big|,
$$
$$
\hat{c_i} = c_i + \Phi(D_i) - \Phi(D_{i-1}).
$$

Only during the **“expansive”** `QUACKPULL` or `QUACKPOP` operations,
the cost is non-trivial:
$$
\hat{c_i} = \frac{3}{2}n + 0 - c n.
$$

When $c > \frac{3}{2}$, we have $\hat{c_i} < 0$.

Therefore, the amortized time per operation is  
$$
O(1).
$$

## Problem 1

**Basic idea:**  
Use a stack to scan through the expression. When encountering numbers or `!`, simply append them to the end of the new postfix expression. When encountering a `+` or `*`, POP all `*` operators from the stack and add them to the postfix expression, then PUSH the current operator onto the stack. After scanning the entire expression, POP all remaining operators from the stack and append them to the postfix expression. The time complexity is $O(n)$.

**Pseudocode:**
```python
getPostfix(Expression exp):
    Stack s
    Expression result
    for ele in exp:
        if ele is '!' or ele is a number:
            result.addBack(ele)
        else: // ele is '+' or '*'
            while s.top is '*':
                result.addBack(s.pop())
            s.push(ele)
    while not s.empty():
        result.addBack(s.pop())
    return result
```
## Problem 2
### (a)

**Guess:** $T(n) \geq dn = \Omega(n)$

**Induction base:** $T(n) = 1 \geq d \cdot 1$ as long as $d \leq 1$.

**Inductive step:**  
$$
T(n) = T\!\left(\tfrac{1}{4}n\right) + T\!\left(\tfrac{3}{4}n\right) 
      \;\geq\; d\!\left(\tfrac{1}{4}n\right) + d\!\left(\tfrac{3}{4}n\right) 
      \;=\; dn
$$
### (b)

- Root node: one node, total cost $\Theta(n)$  
- 1st layer: two nodes, total cost $\Theta(n)$  
- 2nd layer: four nodes, total cost $\Theta(n)$  
- $\cdots$
- $k$-th layer: $2^k$ nodes, total cost $\Theta(n)$  

We assume that the $k$-th layer is the last layer where $2^k$ nodes exist; below this layer, some nodes reach the base case and do not continue recursion.

Assume $\alpha < 1 - \alpha$. Then $k$ should be larger than $\log_{1/(1-\alpha)} n$ and less than $\log_{1/\alpha} n$.

At the layer of $\log_{1/(1-\alpha)} n$, the total time cost is $n \log_{1/(1-\alpha)} n$, so $T(n) = \Omega(n \log n)$.

At the layer of $\log_{1/\alpha} n$, the total time cost is $n \log_{1/\alpha} n$, so $T(n) = O(n \log n)$.

Therefore, $T(n) = \Theta(n \log n)$.

## Problem 3

Having $m = \log n$, we define $S(m) = T(2^m)$ with the recurrence:  

$$
T(n) = 2T(\sqrt{n}) + \Theta(\log n)
$$  

Then we can rewrite it as:  

$$
S(m) = 2S(m/2) + \Theta(m)
$$  

---

### (b)  
Using a recursion tree:  

- Each layer has time cost $\Theta(m)$.  
- The number of layers is $\log m$.  

Therefore,  

$$
S(m) = \Theta(m \log m)
$$  

Since $S(m) = T(2^m)$ and $m = \log n$, we get:  

$$
T(2^m) = T(n) = \Theta(\log n \cdot \log(\log n))
$$  

Thus,  

$$
T(n) = \Theta(\log n \cdot \log \log n)
$$

### (c)  
Define $m = \log n$ and $T(n) = S(\log n)$.  

We have:  

$$
S(\log n) = 3S\!\left(\tfrac{1}{3}\log n\right) + \Theta(n)
$$  

Equivalently,  

$$
S(m) = 3S(m/3) + \Theta(2^m)
$$  

Using the **Master Theorem**, we conclude:  

$$
S(m) = \Theta(2^m)
$$  

Therefore,  

$$
S(m) = T(2^m) = T(n) = \Theta(n)
$$  



## Problem 4

**Basic Idea:**  
- **Divide:** Split the array into two halves of length `n/2`.  
- **Conquer:** For each half, compute the following:
  - total sum  
  - maximum subarray sum  
  - maximum left sum  
  - maximum right sum  
- **Combine:** Merge the results from the two halves to calculate the same four values for the entire array.  
The overall time complexity is **O(n)**.

**Pseudocode:**
```python
maxSub(arr, start, end):
    if end - start == 1:
        return (arr[start], arr[start], arr[start], arr[start])
        # (total sum, left max, right max, total max)
    else:
        mid = (start + end) // 2
        leftResult = maxSub(arr, start, mid)
        rightResult = maxSub(arr, mid, end)

        leftMax = max(leftResult.totalSum + rightResult.leftMax, leftResult.leftMax)
        rightMax = max(rightResult.totalSum + leftResult.rightMax, rightResult.rightMax)
        totalMax = max(leftResult.totalMax,rightResult.totalMax,leftResult.rightMax + rightResult.leftMax)
        totalSum = leftResult.totalSum + rightResult.totalSum

        return (totalSum, leftMax, rightMax, totalMax)
```

## Problem 5
### (a)
if a or b is 1, then gcd is the other one. if not, follow the three rules and do recursion.
```python
gcd(a, b):
    if a == 1: return b
    if b == 1: return a
    else:
        if a is even and b is even:
            return gcd(a/2, b/2)
        if a is even and b is odd:
            return gcd(a/2, b)
        if a is odd and b is even:
            return gcd(a, b/2)
        if a is odd and b is odd:
            return gcd(|a-b|/2, b)
```
### (b)
We assume that m is the binary bit number which coresponds to n in decimal. For each recursion, a or be decreases by at least half. So the depth of the recursion tree is at most m layers. Division is same as shifting, which is O(m) in time. Thus, the total time complexity is O(m^2). which is same as O(n^2).

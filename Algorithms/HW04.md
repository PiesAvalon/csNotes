**Max-Heap** is better. Deletion and heap reorganization can be done in $O(\log n)$ time, while in a sorted array, deleting requires shifting elements, which takes $O(n)$ time.

**Max-Heap** is better. A max-heap can be built in $O(n)$ time, while forming a sorted array requires sorting, which takes $O(n \log n)$ time.

## Bonus Problem

Assume the heap is full, with $2^k - 1$ nodes. When sorting the first half of $2^{k-1} - 1$ nodes, these nodes must form a binary tree. If a node is popped out, its parent must already have been popped out. Because of this, it is easy to prove that at most $2^{k-2}$ nodes can lie in the lowest layer of the original heap.  

Thus, at least $2^{k-2} - 1$ of the $2^{k-1} - 1$ nodes being sorted are not leaves of the original heap. This means that when they are sorted, they are not swapped directly to the root but must "swim up" to the root gradually.  

Now, calculate the sum of the distances from all nodes of a $2^{k-1} - 1$ binary tree to the root. The total distance is:

$$
\sum_{i=1}^{k-2} i \cdot 2^i = 2 \cdot (k-3) \cdot 2^{k-3} + 2 = \Theta(n \log n).
$$

The above calculation is for sorting the first half of the heap. Therefore, sorting the entire heap requires at least:

$$
\Omega(n \log n)
$$

time.

### (a)

First, we prove that **UNUSUAL** correctly combines two sorted arrays, namely $A[1 \dots (n/2)]$ and $A[(n/2) \dots n]$. To prove this, we use induction.  

- **Induction Basis:** When $n \leq 2$, **UNUSUAL** is correct.  
- **Induction Hypothesis:** Assume when $n = k$, **UNUSUAL** is correct.  
- **Induction Step:** When $n = 2k$, since $A[1 \dots (n/2)]$ and $A[(n/2) \dots n]$ are sorted, the largest $n/4$ items of $n$ items must lie in $A[n/4 \dots n/2]$ and $A[3n/4 \dots n]$. After the 2nd and 3rd quarters are swapped, the largest $n/4$ items are guaranteed to be in $A[n/2 \dots n]$, and similarly, the smallest $n/4$ items are in $A[1 \dots (n/2)]$.  
  By the induction hypothesis, **UNUSUAL** correctly sorts $k$ items. After two recursive calls of **UNUSUAL**, both $A[1 \dots (n/2)]$ and $A[(n/2) \dots n]$ are sorted, with $A[1 \dots (n/4)]$ and $A[3n/4 \dots n]$ in the correct positions. After the final call of **UNUSUAL**, $A[n/4 \dots 3n/4]$ is sorted, and thus the entire array of $n$ elements is sorted.  
  Therefore, **UNUSUAL** is correct.  

Now we prove that **CRUEL** correctly sorts $n$ elements.  

- **Induction Basis:** When $n \leq 2$, **CRUEL** is correct.  
- **Induction Hypothesis:** Assume when $n = k$, **CRUEL** is correct.  
- **Induction Step:** When $n = 2k$, the two recursive calls of **CRUEL** correctly sort $A[1 \dots k]$ and $A[k+1 \dots n]$. The final call of **UNUSUAL** combines the two sorted subarrays, whose correctness has already been proven.  
  Therefore, **CRUEL** is correct.

## Problem 3

### (a)

No matter what position the smallest element is in, assume it is at position $i$. Executing $\text{REVERSE}(p, 1, i)$ places the smallest element at the front. Then, similarly, place the 2nd smallest element at the second position, and continue this process iteratively. In this way, the list of items will be sorted in $O(n)$ operations.

---

### (b)

Define two adjacent elements that are not consecutive in value as a **breakpoint**. In a sorted array, the number of breakpoints is zero. A single **REVERSE** operation can reduce at most two breakpoints. Therefore, in a permutation with $n-1$ breakpoints (e.g., a completely reversed array), it would take at least

$$
\frac{n-1}{2} = \Theta(n)
$$

**REVERSE** operations to eliminate all breakpoints. Thus, achieving $o(n)$ operations is impossible.

---

### (c)

#### Basic Idea

Similar to Merge Sort, we define an algorithm called **SORT**.  

1. **Divide:** Split array $p$ into two equally sized subarrays $A$ and $B$.  
2. **Recursive Sort:** Call **SORT** to sort each subarray.  
3. **Combine:** Merge $A$ and $B$ using a divide-and-conquer strategy.  

#### Combine Step

- Divide $A$ into two equal subarrays $A_1$ and $A_2$, where all elements in $A_1$ are less than a pivot $k$, and all elements in $A_2$ are greater than $k$.  
- Perform a binary search on $B$ to find a breakpoint that separates $B$ into $B_1$ and $B_2$, where all elements in $B_1 < k$ and all elements in $B_2 > k$.  
- Use **ROTATE** to swap $A_2$ and $B_1$.  

Now the problem reduces to two smaller combine tasks: merging $(A_1, B_1)$ and $(A_2, B_2)$.  

#### Rotate Step

The operation $\text{ROTATE}(X, Y)$ can be performed using three reversals:

1. Reverse $X$.  
2. Reverse $Y$.  
3. Reverse $(X^r Y^r)$.  

This costs $O(|X| + |Y|)$ time.  

#### Complexity

At each level of the **COMBINE** process, $O(n)$ work is done. Since the recursion depth is $O(\log n)$, the total cost of **COMBINE** is:

$$
O(n \log n).
$$

Thus, the recurrence relation for **SORT** is:

$$
T(n) = 2T(n/2) + O(n \log n).
$$

By the Master Theorem, the total runtime of **SORT** is:

$$
T(n) = O(n \log^2 n).
$$

#### Pseudocode
```python
ROTATE(p, A, B):
    REVERSE(p, index of A start, index of A end)
    REVERSE(p, index of B start, index of B end)
    REVERSE(p, index of A start, index of B end)

COMBINE(A, B):
    A1, A2 = divide A into two subarraies of same length where A1 < k, A2 > k # O(1) time
    B1, B2 = divide B into two subarraies where B1 < k, B2 > k # binary search O(logn) time
    ROTATE(A2, B1) # O(n) time
    COMBINE(A1, B1)
    COMBINE(A2, B2)

SORT(p):# O(nlog^2n time)
    if p.len == 1:
        return
    A, B = divide p into two subarraies of same length
    SORT(A)
    SORT(B)
    COMBINE(A, B) # O(nlogn) time

```
#### Proof

First, we prove that **COMBINE** is correct.  

- **Induction Basis:** When $|A| = |B| = 1$, it is obvious that **COMBINE** is correct.  
- **Induction Hypothesis:** Assume that when $|A| = |B| \leq k$, **COMBINE** is correct.  
- **Induction Step:** When $|A| = |B| = k + 1$, after the **ROTATE** operation, the subarrays $(A_1, B_1)$ and $(A_2, B_2)$ both have length at most $k$.  
  The construction of $A_1, B_1, A_2, B_2$ guarantees that the largest element in $(A_1, B_1)$ is smaller than the smallest element in $(A_2, B_2)$, and $A_1, B_1, A_2, B_2$ are all sorted subarrays.  
  Thus, the entire **COMBINE** is reduced to two smaller **COMBINE** problems. Therefore, the algorithm is correct.  

---

Next, we prove that **SORT** is correct.  

- **Induction Basis:** When $|p| = 1$, **SORT** is trivially correct.  
- **Induction Hypothesis:** Assume that when $|p| \leq k$, **SORT** is correct.  
- **Induction Step:** When $|p| = k + 1$, the problem is divided into two **SORT** problems of size at most $k$.  
  Since each of these is correctly sorted by the induction hypothesis, and their results are correctly merged by **COMBINE** (already proven), the entire **SORT** algorithm is correct.  


## Problem 4

### Basic Idea

Create a new array and iterate through the array of items (which is sorted by number) three times.  
- In the first iteration, place all red items at the back of the new array.  
- In the second iteration, place all blue items.  
- In the third iteration, place all yellow items.  

Since there are only parallel `for` loops, the total time cost is:

$$
O(3n) = O(n).
$$


### Pseudocode

```python
sortByColor(array a):
    array newArray
    for item in a:
        if item.color is red:
            newArray.addlast(item)
    for item in a:
        if item.color is blue:
            newArray.addlast(item)
    for item in a:
        if item.color is yellow:
            newArray.addlast(item)
    return newArray
```

## Problem 5
### Basic Idea

Create an array $a$ of length $n+1$. Scan through the array of papers: for each paper, if its number of citations is $k$, increment $a[k]$ (if $k > n$, increment $a[n]$ instead).  

After this, scan array $a$ backwards, maintaining the cumulative sum of $a[i] + a[i+1] + \dots + a[n]$ until the index $i$ is less than this sum.  

Since we only iterate through arrays of length $n$, the total time cost is:

$$
O(n).
$$

### Pseudocode
```python
hIndex(array papers):
    array a # size if paper.len + 1, all elements are inited 0
    for paper in papers:
        a[min(paper.citations, n)] += 1
    papernum = 0
    for k = n to k = 0:
        papernum += a[k]
        if k <= papernum:
            return k
```

## Problem 6
### (a)
#### Basic Idea

Use a fast sorting algorithm to sort both arrays. Then, apply a merge-like algorithm to combine the two sets.  

- The sorting step costs $O(n \log n)$ time.  
- The merging step costs $O(n)$ time.  

Thus, the total cost is:

$$
O(n \log n).
$$

It is assumed that sets $A$ and $B$ contain no duplicate elements.

#### Pseudocode
```python
union(array A, array B):
    heapSort(A)
    heapSort(B)
    array U
    i = j = 0
    while i < A.len and j < B.len:
        if A[i] < B[j]:
           U.add(A[i])
           i += 1
        else if A[i] == B[j]:
           U.add(A[i])
           i += 1
           j += 1
        else if A[i] > B[j]:
            U.add(B[j])
            j += 1
    if i == A.len:
        U.add(B[j...B.len-1])
    else:
        U.add(A[i...A.len-1])
    return U
```
